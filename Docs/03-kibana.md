# Phase 3 - Kibana Installation

## Objective
Deploy and configure Kibana as the web interface for the Elastic Stack, enabling visualization and management of data stored in Elasticsearch.

## Implementation

### Install Kibana
Kibana was installed using the official Elastic APT repository that was configured during the Elasticsearch deployment. Installing Kibana from the official Elastic APT repository allows the application to be managed through Ubuntu's package manager (apt), simplifying future updates and maintenance.

The installation created the core components required to operate Kibana, including:

- The Kibana application
- A dedicated `kibana` system user and group
- A `systemd` service for managing Kibana
- Configuration files located in `/etc/kibana`
- Log files stored in `/var/log/kibana`

### Configure Kibana
By default, Kibana is configured to listen only on the local loopback interface (`localhost`). Since Kibana would be accessed remotely from another machine on the local network, the `server.host` setting was changed to `0.0.0.0` within the `/etc/kibana/kibana.yml` file.

This configuration allows Kibana to accept connections on all network interfaces while remaining accessible only within the local network.

### Enable the Service
The `kibana.service` unit was enabled to ensure Kibana automatically starts whenever the Ubuntu server boots. The service was then started to launch Kibana immediately without requiring a system reboot.

### Additional Configuration
After Kibana became accessible through the web interface, it was enrolled with the Elasticsearch instance using an enrollment token generated by Elasticsearch. The enrollment process established a trusted relationship between Kibana and Elasticsearch, allowing the two components to communicate securely.

The enrollment token was generated from the Elasticsearch server using:
```
sudo /usr/share/elasticsearch/bin/elasticsearch-create-enrollment-token -s kibana
```

Following the successful enrollment, the password for the built-in `elastic` superuser account was reset to a custom password for easier administration of the homelab environment.

The password was reset using:
```
sudo /usr/share/elasticsearch/bin/elasticsearch-reset-password -i -u elastic
```

## Key Takeaways
- Kibana serves as the web interface for Elasticsearch and does not store log data itself.
- Kibana listens only on `localhost` by default and must be configured to listen on another interface to allow remote access.
- Services must be restarted after configuration changes are made for the changes to take effect.
- Successfully running services do not always indicate that an application is reachable over the network; network accessibility and configuration should also be verified.
- The Kibana enrollment process establishes a secure, trusted connection with Elasticsearch using an enrollment token rather than requiring manual certificate configuration.
